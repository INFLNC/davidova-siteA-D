# Davidova.Studio — инструкция по обновлению

## 1. Как устроены ссылки (важно — это причина проблемы с D:\)

**Абсолютный путь** — `D:\davidova-site\assets\img_EGO\Hero.png`
Работает ТОЛЬКО на твоём компьютере. На хостинге диска D: нет → картинки не грузятся.

**Относительный путь** — `assets/img_EGO/Hero.png`
Работает везде. Означает «ищи рядом со мной».
- `assets/...` — от текущей папки вниз
- `../assets/...` — на папку вверх, потом вниз (`..` = «родительская папка»)

**Правило в этом проекте:**
| Файл лежит в | Пишет путь к картинке |
|---|---|
| корень (`index.html`, `work.html`) | `assets/img_EGO/Hero.png` |
| папка `work/` (`ego.html` и др.) | `../assets/img_EGO/Hero.png` |

В новых файлах **нет ни одного `D:\`** — всё относительное. Картинки подхватятся сами, как только лежат в правильных папках `assets/`.

Положи `hover.mp4` в папку проекта → при наведении на карточку статичный рендер сменяется видео. Нет файла — остаётся静 статика, без ошибок.
Конвертация: `ffmpeg -i in.mov -vcodec h264 -an -vf scale=1280:-2 -crf 28 -movflags +faststart hover.mp4`

## 4. Мобильная версия — как работает

`mobile.css` подключён в каждой странице через:
```html
<link rel="stylesheet" href="mobile.css" media="(max-width: 880px)">
```
(в папке `work/` — `href="../mobile.css"`)

**Картинки НЕ дублируются.** Мобильная версия берёт те же файлы из `assets/`, что и десктоп. Отдельные папки под мобилку создавать НЕ нужно — экономит место и время.

Что делает mobile.css: одна колонка вместо двух, компактное меню, фильтры скроллятся вбок, hover-видео отключено (на тач-экранах нет наведения — показывается статичный рендер).

## 5. Стиль вынесен в отдельный CSS — как это работает

`assets/style.css` — единый источник стиля для главной, showcase и about-разметки. Один файл правит весь «каркас»: цвета, шрифты, шапку, карточки, футер.

Хочешь поменять акцентный цвет на всём сайте? Открой `style.css`, найди `--accent: #ff2e16`, поменяй — обновится везде разом.

Case study (ego, petro, aoi) имеют стиль внутри себя (inline `<style>`), потому что у них уникальная вёрстка. Это нормально — они самодостаточны.

## 6. Почему сайт пока показывает старый стиль

Браузеры кешируют CSS. После заливки новых файлов:
1. Открой сайт и нажми **Ctrl+Shift+R** (жёсткая перезагрузка без кеша)
2. Проверь, что `index.html` действительно заменён на хостинге (дата файла свежая)
3. Если хостинг с CDN (Cloudflare и т.п.) — очисти кеш в его панели
